# 🚀 Frontend Engineering Handbook

## 📌 Overview

This repository is a **professional Frontend Engineering Handbook** designed to standardize how scalable, maintainable, and production-ready frontend applications are built using:

- React 19
- TypeScript
- Vite
- Redux Toolkit
- RTK Query
- Tailwind CSS
- shadcn/ui

It follows **Clean Architecture, Feature-Based Structure, and Enterprise-Level Best Practices**.

---

## 🎯 Purpose

The goal of this handbook is to:

- Define a **unified frontend architecture standard**
- Enforce **clean, scalable, and maintainable code**
- Provide **reusable patterns and templates**
- Help developers (and AI) build **production-ready applications**
- Avoid bad practices, spaghetti code, and duplication

---

## 🧠 Who is this for?

- Frontend Developers (Junior → Senior)
- React Developers
- Teams building scalable applications
- AI-assisted development workflows

## ⚙️ How to use this handbook

1. Start by reading:
   - `docs/01-project-philosophy.md`
   - `docs/02-ai-development-rules.md`

2. Follow rules in order.

3. Use templates when creating new features.

4. Check checklists before commit/deploy.

5. Always follow AI rules for consistent code generation.

## 🧱 Core Principles

This handbook is built on:

- Clean Code
- SOLID Principles
- DRY (Don't Repeat Yourself)
- KISS (Keep It Simple)
- YAGNI (You Aren’t Gonna Need It)
- Feature-Based Architecture
- Reusable Components
- Separation of Concerns
