# 📖 Glossary

A concise reference of the core frontend engineering terms used throughout this handbook. Each entry explains *what* the term means, *why* it matters, and *how* it appears in this project.

| Term | One-line definition |
| --- | --- |
| [Hydration](terms.md#hydration) | Attaching React interactivity to server-rendered HTML |
| [Memoization](terms.md#memoization) | Caching a computed result to avoid recomputation |
| [Re-render](terms.md#re-render) | React re-running a component after state/prop change |
| [CSR](terms.md#csr-client-side-rendering) | Rendering in the browser via JavaScript |
| [SSR](terms.md#ssr-server-side-rendering) | Rendering HTML on the server per request |
| [SPA](terms.md#spa-single-page-application) | One HTML shell, client-side navigation |
| [Lazy Loading](terms.md#lazy-loading) | Loading code/resources only when needed |
| [Tree Shaking](terms.md#tree-shaking) | Removing unused code from the bundle |
| [Bundle Splitting](terms.md#bundle-splitting) | Splitting output into smaller loadable chunks |
| [Closure](terms.md#closure) | A function retaining access to its outer scope |
| [Hoisting](terms.md#hoisting) | JS moving declarations to the top of their scope |

See the full definitions in [terms.md](terms.md).
